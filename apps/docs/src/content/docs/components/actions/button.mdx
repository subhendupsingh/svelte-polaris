# Button

Buttons are used to initialize an action, such as submitting a form, cancelling an action, or performing a page specific operation. Buttons can also be used for navigation when styled as a link.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'plain' \| 'primary' \| 'secondary' \| 'tertiary' \| 'monochromePlain'` | `'secondary'` | Changes the visual appearance of the button |
| `tone` | `'critical' \| 'success'` | - | Sets the color treatment of the button |
| `size` | `'micro' \| 'slim' \| 'medium' \| 'large'` | `'medium'` | Changes the size of the button |
| `textAlign` | `'left' \| 'right' \| 'center' \| 'start' \| 'end'` | `'start'` | Changes the inner text alignment |
| `fullWidth` | `boolean` | `false` | Allows the button to grow to the width of its container |
| `disabled` | `boolean` | `false` | Disables the button |
| `loading` | `boolean` | `false` | Shows loading state with spinner |
| `pressed` | `boolean` | `false` | Indicates the pressed state for toggle buttons |
| `disclosure` | `'down' \| 'up' \| 'select' \| boolean` | `false` | Displays the button with a disclosure icon |
| `icon` | `IconSource` | - | Icon to display to the left of the button content |
| `url` | `string` | - | A destination to link to, rendered in the href attribute of a link |
| `external` | `boolean` | `false` | Forces url to open in a new tab |
| `download` | `boolean \| string` | `false` | Tells the browser to download the url instead of opening it |
| `accessibilityLabel` | `string` | - | Visually hidden text for screen readers |
| `onClick` | `() => void` | - | Callback when button is clicked |

## Basic Usage

### Default Button

The default button for most actions.

```svelte
<script>
  import { Button } from 'svelte-polaris';

  function handleClick() {
    console.log('Button clicked');
  }
</script>

<Button onclick={handleClick}>
  Default button
</Button>
```

### Primary Button

Primary buttons are used for the most important actions.

```svelte
<script>
  import { Button } from 'svelte-polaris';
</script>

<Button variant="primary">
  Save changes
</Button>
```

### Secondary Button

Secondary buttons are used for secondary actions.

```svelte
<script>
  import { Button } from 'svelte-polaris';
</script>

<Button variant="secondary">
  Cancel
</Button>
```

## Buttons with Icons

### Icon Only Button

Use icon-only buttons when space is limited and the icon meaning is clear.

```svelte

  import { SearchIcon, DeleteIcon, EditIcon } from 'svelte-polaris/icons';
</script>

<Button accessibilityLabel="Search" icon={SearchIcon} />
<Button accessibilityLabel="Delete" icon={DeleteIcon} tone="critical" />
```

## Complex Examples

### Form Actions

Common pattern for form submission and cancellation.

```svelte
<script>
  import { Button, ButtonGroup } from 'svelte-polaris';
  
  let isSubmitting = false;
  
  async function handleSubmit() {
    isSubmitting = true;
    try {
      // Submit form logic
      await submitForm();
    } finally {
      isSubmitting = false;
    }
  }
  
  function handleCancel() {
    // Cancel logic
  }
</script>

<ButtonGroup>
  <Button onclick={handleCancel}>
    Cancel
  </Button>
  <Button variant="primary" loading={isSubmitting} onclick={handleSubmit}>
    Save product
  </Button>
</ButtonGroup>
```

### Action with Success State

Example showing state management for async actions.

```svelte
<script>
  import { Button } from 'svelte-polaris';
  
  let loading = false;
  let success = false;
  
  async function handleAction() {
    loading = true;
    try {
      await performAction();
      success = true;
      setTimeout(() => success = false, 2000);
    } finally {
      loading = false;
    }
  }
</script>

{#if success}
  <Button variant="primary" tone="success" disabled>
    âœ“ Saved successfully
  </Button>
{:else}
  <Button variant="primary" {loading} onclick={handleAction}>
    Save changes
  </Button>
{/if}
```

### Destructive Action with Confirmation

Pattern for dangerous actions that require user confirmation.

```svelte
<script>
  import { Button } from 'svelte-polaris';
  
  let showConfirmation = false;
  
  function handleDelete() {
    if (showConfirmation) {
      // Perform delete action
      console.log('Item deleted');
      showConfirmation = false;
    } else {
      showConfirmation = true;
      // Auto-hide confirmation after 3 seconds
      setTimeout(() => showConfirmation = false, 3000);
    }
  }
</script>

<Button 
  tone="critical" 
  variant={showConfirmation ? "primary" : "secondary"}
  onclick={handleDelete}
>
  {showConfirmation ? "Confirm delete" : "Delete product"}
</Button>
```

## Accessibility

- Always provide descriptive text or `accessibilityLabel` for screen readers
- Use semantic `tone` props to convey meaning (critical for destructive actions)
- Ensure sufficient color contrast for all button variants
- Consider focus states and keyboard navigation
- For icon-only buttons, always include `accessibilityLabel`
